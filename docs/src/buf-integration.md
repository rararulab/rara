# Buf Integration Guide

This project uses [Buf](https://buf.build/) for proto linting/breaking checks and Python gRPC code generation for the execution worker.

## Current Generation Strategy

- Rust gRPC/proto code is generated by `/Users/ryan/code/personal/rust/job/api/build.rs` using `tonic-prost-build` during Cargo builds.
- Buf code generation is used only for Python stubs needed by the execution worker.
- Generated Python files are written to `/Users/ryan/code/personal/rust/job/api-generated/python/pb`.

## Config Files

- `api/buf.yaml`: Buf module + lint/breaking config
- `api/buf.gen.yaml`: Python generation config (execution worker proto only)
- `api/buf.lock`: Buf dependency lockfile

## Commands

```bash
# lint / format / breaking checks
cd /Users/ryan/code/personal/rust/job/api
buf lint
buf format -w
buf breaking --against .git#branch=main

# generate Python execution worker stubs only
buf generate
```

## What `buf generate` Produces

`api/buf.gen.yaml` is intentionally scoped to:

- `proto/execution/v1/worker.proto`
- Python protobuf + gRPC stubs only

Output files:

```text
api-generated/
  python/
    pb/
      execution/
        v1/
          worker_pb2.py
          worker_pb2_grpc.py
```

## Notes

- If you add new execution worker protos, update `api/buf.gen.yaml` `inputs.paths`.
- If you add non-Python generated clients, add a separate template intentionally instead of expanding the default one.
