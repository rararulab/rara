# Chat TUI Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Refactor `rara chat` from a line-oriented REPL into a standalone ratatui chat interface that reuses the existing kernel/session plumbing.

**Architecture:** Extract chat-specific bootstrapping and session helpers from `crates/cmd/src/main.rs` into a dedicated `crates/cmd/src/chat/` module. Build a small chat state machine that consumes existing `CliEvent` messages from `TerminalAdapter`, renders them in a ratatui layout, and sends user input back through the kernel using the same CLI channel/session binding model.

**Tech Stack:** Rust, Tokio, crossterm, ratatui, clap, existing `rara-app` / `rara-kernel` / `rara-channels` workspace crates.
