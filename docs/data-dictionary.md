# Data Dictionary

Domain model and database schema reference for the Job Automation Platform.

## Overview

The platform uses PostgreSQL with the following core entities:

| Entity | Table | Purpose |
|--------|-------|---------|
| Job | `job` | Job posting info, source, lifecycle |
| Resume | `resume` | Resume versions, content hashing, generation source |
| Application | `application` | Application records linking jobs to resumes |
| Application Status History | `application_status_history` | Immutable status change audit trail |
| Interview Plan | `interview_plan` | Interview prep tasks and materials |
| Prompt Template | `prompt_template` | AI prompt template library by kind/version |
| AI Run | `ai_run` | Model call records with I/O summary, tokens, cost |
| Notification Log | `notification_log` | Telegram/Email/Webhook notification records |
| Metrics Snapshot | `metrics_snapshot` | Periodic stats snapshots (daily/weekly/monthly) |

## Design Principles

- **Idempotent key**: `source_job_id + source_name` uniquely identifies a job across sources.
- **Soft delete**: All mutable entities have `is_deleted` (boolean) and `deleted_at` (timestamp) columns. Queries should filter on `is_deleted = FALSE` by default.
- **Full-chain timestamps**: Every entity has `created_at`; mutable entities also have `updated_at` (auto-managed by a database trigger).
- **Trace ID**: Every entity carries an optional `trace_id` field for distributed tracing correlation.
- **JSONB flexibility**: Entities with evolving schemas use `JSONB` columns (`raw_data`, `metadata`, `materials`, `extra`) for forward compatibility.

## Tables

### `job`

Job posting discovered from an external source.

| Column | Type | Nullable | Default | Description |
|--------|------|----------|---------|-------------|
| `id` | `UUID` | No | `gen_random_uuid()` | Primary key |
| `source_job_id` | `TEXT` | No | | External ID from source platform |
| `source_name` | `TEXT` | No | | Source platform name (e.g. "linkedin") |
| `title` | `TEXT` | No | | Job title |
| `company` | `TEXT` | No | | Company name |
| `location` | `TEXT` | Yes | | Job location |
| `description` | `TEXT` | Yes | | Full job description |
| `url` | `TEXT` | Yes | | Link to original posting |
| `salary_min` | `INTEGER` | Yes | | Minimum salary |
| `salary_max` | `INTEGER` | Yes | | Maximum salary |
| `salary_currency` | `TEXT` | Yes | | Salary currency code |
| `tags` | `TEXT[]` | No | `'{}'` | Searchable tags |
| `status` | `job_status` | No | `'active'` | Lifecycle status |
| `raw_data` | `JSONB` | Yes | | Raw source data |
| `trace_id` | `TEXT` | Yes | | Distributed trace ID |
| `is_deleted` | `BOOLEAN` | No | `FALSE` | Soft delete flag |
| `deleted_at` | `TIMESTAMPTZ` | Yes | | Soft delete timestamp |
| `posted_at` | `TIMESTAMPTZ` | Yes | | When job was posted on source |
| `created_at` | `TIMESTAMPTZ` | No | `now()` | Record creation time |
| `updated_at` | `TIMESTAMPTZ` | No | `now()` | Last update time (auto-trigger) |

**Unique constraint**: `(source_job_id, source_name)` -- idempotent key

**Indexes**: `title`, `company`, `location`, `status`, `updated_at`, `source_name`, `is_deleted` (partial)

## Entity Relationships

```
job 1---* application *---1 resume
              |
              |--- application_status_history (append-only)
              |
              *--- interview_plan

prompt_template 1---* ai_run

notification_log (references any entity via reference_type + reference_id)

metrics_snapshot (standalone aggregation, no FK dependencies)
```

## Rust Model Location

All domain model structs live in `crates/common/yunara-store/src/models/`:

| File | Structs / Enums |
|------|-----------------|
| `job.rs` | `Job`, `JobStatus` |
| `resume.rs` | `Resume`, `ResumeSource` |
| `application.rs` | `Application`, `ApplicationChannel`, `ApplicationStatus`, `ApplicationStatusHistory` |

| `notification.rs` | `NotificationLog`, `NotificationChannel`, `NotificationStatus` |
